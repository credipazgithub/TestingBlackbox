Información del cliente basada en su DNI

Devuelve los datos de un cliente de acuerdo a los datos provistos y dado el token_authentication y id_user_active autenticado.

Endpoint
content_copy
Parámetros
Token string requerido
ID usuario activo integer requerido
ID application integer requerido
DNI integer requerido
Sexo string

Valores permitidos: M/F

Si no se envía o se envía cadena vacía, no filtra por sexo.

 opcional
Email string requerido
Área integer requerido
Teléfono integer requerido
Ejecutar
Observaciones

Para integraciones externas se debe utilizar 11, como valor de ID application, el cual está fijo en estos ejemplos.

Debe enviarse el token_authentication y el id_user_active, ambos valores obtenidos por medio del endpoint /production/authenticate.
